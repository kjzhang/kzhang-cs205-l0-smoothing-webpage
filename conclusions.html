Conclusions

Our project demonstrates a fully parallel implementation of a complex algorithm that relies on several dependencies, outside packages, and additional complications. We implemented both the serial and parallel versions, which allows us to compare performance across a variety of image sizes under the same infrastructure and environment. By using PyCUDA, we were able to achieve a tremendous speedup, making the algorithm accessible for larger images, and perhaps in the future, video transformations.

The work we did on this project gave us a new appreciation for the difficulty of implementing complex algorithms in parallel. We had several severe struggles with the various dependencies and packages that we had to use to implement the algorithm. Furthermore, in our initial trials, the performance was not as strong as we would have liked, which forced us to parallelize essentially the entire algorithm. There was often times where the bottleneck and the least efficient part of the algorithm turned out to be the hardest part to parallelize, which made it difficult. There are definitely more improvements we can make, such as implementing the algorithm in a faster language, such as C++, or parallelizing the helper functions, but so far we have achieved a substantial speedup through parallelization. If we have time in the future, we may try to implement a parallel C++ version to achieve even better performance.

We both definitely learned a lot throughout this process. It was a really interesting project, taught us more about how parallel development works in a less structured environment (not a problem set) and forced us to hack our way through tough technical problem. We want to thank our TF Brandon for all the help and guidance along the way, as well as Professor Pfister and Cecka for their valuable lectures and problem sets.
